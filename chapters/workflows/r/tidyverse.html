9  Data Wrangling with tidyverse

The tidyverse is a collection of R packages designed for data science that share a common philosophy and syntax. Rather than learning many unrelated packages, the tidyverse gives you a coherent set of tools that work well together. Load it with:

library(tidyverse)

This loads several packages at once, including dplyr for data manipulation and ggplot2 for visualization.

9.1 Reading Data

Use read_csv() to read a CSV file into R:

library(tidyverse)
library(here)

data <- read_csv(here("data", "cleaned", "wave1_cleaned.csv"))

The here() function constructs file paths relative to the project root, so your code works regardless of where the project is stored on someone’s computer. Always use here() rather than hardcoded paths.

For Excel files, use the readxl package:

library(readxl)

data <- read_excel(here("data", "raw", "survey_data.xlsx"))

For Stata files, use the haven package:

library(haven)

data <- read_dta(here("data", "raw", "survey_data.dta"))

9.2 Viewing Data

After loading data, get a quick overview:

# Print the first few rows
head(data)

# See dimensions (rows and columns)
dim(data)

# See variable names
names(data)

# See variable types and a preview
glimpse(data)

# Summary statistics for all variables
summary(data)

9.3 Manipulating Data with dplyr

dplyr provides a set of functions for common data manipulation tasks. These functions are designed to be used with the pipe operator |>, which passes the output of one function as the input to the next:

Select columns:

data |>
  select(student_id, grade, sel_total_mean)

Filter rows:

data |>
  filter(grade == 3)

data |>
  filter(grade >= 3 & sel_total_mean > 2.5)

Create or recode variables:

data |>
  mutate(
    sel_total_z = scale(sel_total_mean),
    grade_cat = if_else(grade <= 2, "lower", "upper")
  )

Summarize data:

data |>
  summarise(
    n = n(),
    mean_sel = mean(sel_total_mean, na.rm = TRUE),
    sd_sel = sd(sel_total_mean, na.rm = TRUE)
  )

Group and summarize:

data |>
  group_by(grade) |>
  summarise(
    n = n(),
    mean_sel = mean(sel_total_mean, na.rm = TRUE)
  )

9.4 Handling Missing Data

Before any analysis, recode CWER missing data codes to NA:

data <- data |>
  mutate(across(everything(), ~ if_else(. %in% c(-99, -98, -97, -96), NA, .)))

Always check for missing values after recoding:

data |>
  summarise(across(everything(), ~ sum(is.na(.)))) |>
  pivot_longer(everything(), names_to = "variable", values_to = "n_missing") |>
  filter(n_missing > 0)

Flag any variables with more than 10% missing for discussion with Mike or Joanna before proceeding with analysis.

9.5 Saving Data

Save cleaned datasets to data/cleaned/ using write_csv() for cross-software compatibility, or saveRDS() to preserve R-specific attributes:

# For sharing across software
write_csv(data_cleaned, here("data", "cleaned", "wave1_cleaned.csv"))

# For use within R only (preserves factors, labels, etc.)
saveRDS(data_cleaned, here("data", "cleaned", "wave1_cleaned.rds"))

Use write_csv() when the cleaned data will be read by Stata, SPSS, or Mplus. Use saveRDS() when the data will only be used in R — it is faster and preserves variable types exactly.
